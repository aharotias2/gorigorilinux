Mac OSの思ひ出

2000年ごろの話ですが、私の父が漫画家なので、昔、Mac OS 8を使っていました。なんでもGUIがハードに直結していてマウス・ペンタブレットの反応が良いとかなんとか。

しかしMac OS Xが出ると、父はもうMacはやめてWindowsにするということになり、私が父の使わなくなったiMacを譲り受けました。緑のスケルトンのやつです。

私も昔は絵を描いたのですが、どちらかというとプログラミングの方に興味を持つようになりました。

当時のMacにはHyperCardというアプリがありました。これはスタックというカードを重ねた形のウィンドウにボタンなどを配置し、そのボタンにスクリプトを書くことで機能を持たせることができるという面白いアプリでした。当時HTMLしか書けず、JavaScriptが受け付けなかった私がこれで始めてプログラミングの面白さに気付きました。

しかしHyperCardはちょっとできることが限られており、C言語によるプラグインが開発されていたので、自分もと思いC言語を始めたのですが、あまりの難しさに一旦退却しました。

その後MacPascalという今で言うIDE (統合開発環境)のようなプログラミングを学べるツールが無料でネットに放流されていたのでそれを使ってPascalという言語を学び、C言語をやった時には分からなかったモヤモヤがどんどん晴れていきました。思うにC言語は自由過ぎて初心者にはどう書けば良いのか分かりづらいのですが、Pascalは、こう書けば良い、ということを教えてくれる言語だったんです。また、MacPascal自体は大したことはできないのですが、GUIのデバッガがあり、変数の変化を見られるのでとても分かりやすかったです。

更にAppleが昔開発環境として使っていたMPWも旧Mac開発中止に伴い無料化してネットに放流されていたのでこれを拾ってC言語を使ってMacのGUIのプログラミングをして遊んでいました。

MacのGUIのプログラミングはとても簡単で面白かったです。まず、旧Macはマルチタスクができないのが欠点とされ、疑似マルチタスクと言われていました。それがMac OS XでUnixベースに転換する主な理由となったのですが、逆に言うとアプリの開発には普通のHello World系のCLIプログラミングと同じように一本道プログラミングができました。main関数から入ってウィンドウなどを出したら、イベントを受けとり、それをswitch文で分岐するのを繰り返すという超シンプルな書き方だったので難しいことは考えずにGUIプログラミングができたのです。グラフを描いたりアニメーションを作るというのもかなり簡単にできました。Windowsだといきなりコールバックとか言う面倒臭そうな仕組みやハンガリアン記法というげんなりする習わし、あと何なんでしょう、WinMainって。意味分かりませんので、好きになれず今だにアプリ一つ作ったことがありません。JavaのSwingが恐らくMacのAPIを真似して作られているので似ていると思いますが、Swingは見栄えがOSに馴染まないのでモチベーションが上がらないという困った特徴がありまして全然使ったことがありません。

余談ですがLinuxについて言うとGTK+プログラミングはWindows以上にめちゃくちゃ面倒臭いです。Valaがないと無理です。まあValaのおかげで大分良くなりましたが。Valaを使うとGTKのコンセプトがハッキリ分かって(デザインパターンは詳しくないですが、オブザーバーパターンが似てるんですかね)、シグナルプログラミングがめちゃ面白く感じるようになるのですが、Cでは超面倒臭いですね。そんだけ苦労してもオリジナリティを出すにはGTK+だけでは駄目でより低レベルAPIのCairoとか使うんでしょうかね。もっと深掘りするにはXlib直打ちですかね。まあエラい大変ですわ。とても無料じゃ割に合わんでしょう。だからあまりアプリがないのです。一説にはGTK開発にはマイクロソフトの社員が関わっていてLinuxの発展を阻むためにわざと難しくしたという陰謀説もあるほどです。その証拠にGTKの公式サイトに見えるウネウネとWindows NT Server 4.0のロゴに見えるウネウネが何か似てる…だそうです。実は皆知ってることだったりするんですかね？たしかにマイクロソフトが関わっていたなら、できることが違ってくるでしょう。マイクロソフトでこき使われてたまったストレスを趣味のGTK開発で発散していた社員がいたとかならホッコリ話しですがね。そんなに大変ですけど、それでも作って無料で公開している人々がいるのは驚くべきことです。Qtは使ったことがないですがね。Qtのほうが見栄えのするアプリを作れるようですが、よくわかりません。(ただ最近のFedoraを見るとGTKも良くなってきてると思いました。あとElectronというのも流行りつつありますね。あれが希望の光となると良いのですが)

まあそんな訳でMacを使ってプログラミングするのは楽しかったですね。今でもMacPascalへの感謝を忘れていません。

しかしやはり時代の流れには逆らえませんでした。当時のMacで使えるウェブブラウザはInternet Explorer 8かNetScape Navigatorがメジャーでしたが、どっちも微妙だったのでiCabというドイツの個人制作ブラウザを使っていたのですが、これがMac OS X対応のため、旧Mac用が開発が中止になってしまいました。当時2ちゃんねるの旧Mac板ではこれを悲しんで有志が頑張って英語で嘆願書を書いて送ったが結局叶わなかったなどの悲しい情景を見て、もう旧Macは終わりだなぁ…と諦めざるを得ないことになりました。私などよりずーーーっとコダワリのあるユーザは意地でも手放さないって感じで、今でも使ってる人もいそうな勢いでしたが、まあぶっちゃけそこまででもなかったのですわ…。

そんな色々あって今でもMac OS Xを使わないのは何となくそういう切り捨て方が気に食わないというのもありますが、一番の理由は単に金が無かったというだけのことです。昔、ニートだったので、Mac OS Xなんか買える訳がありません。

ただiPodは好きでした。まああれがスティーブ・ジョブズの逆襲の始まりでしたな。iTunesの影響力も計り知れなかった。傍目に見ながら、あんなスティーブでも、一度は自分が作ったAppleから追い出されていたなんて、アメリカってやべーなと思ってました。ホンダが本田宗一郎を叩き出すようなものでしょ。そんなことがあるのか。アメリカの文化なんでしょうか？全く理解できませんが凄いですな。

この人たちも私のような社畜じゃないということなんでしょうかね…いくらブラック企業だったとしても社長を追い出すという発想にはなかなかならないような気がします(最近ゴーンさんがすごいことになってましたが)。その昔、勝海舟がアメリカに渡って見聞きしたことを幕府に報告する際、向うでは老害の靴を舐めるようなことは一切ありませんと言ってわしらのことを言っておるのかと老中を激怒させたというエピソードを思い出します…。やっぱ昔からそうなんですな。しかし勝海舟はちゃんとそういうつもりではなかった、誤解だって言ったんですかねぇ、どうやって場を収めたんでしょ。

というわけで、子供部屋から世界の激動を眺めてきましたが、Mac OS Xには縁がありませんでしたな。まあここで何十万もするiMac買ってもドヤ顔できませんし、他にやらなきゃいけないこと山ほどあるので仕方ないですわ…。もし自分に息子、娘がいれば、買ってあげるのも一つの愛の示し方かな…とぼんやり思いますがね。自分に買っても自分にバレンタインチョコあげてるみたいだし。おっとこれは言い過ぎました。取り消します。ごめんなさい。好きならそれで全然問題ないと思います。

皮肉なことに、私が好きなMacってジョブズがいなかったころのものなんですよね。HyperCardなんて、ユーザに自由を与えたくないジョブズ思想の敵だったわけですし。ということはジョブズがいない今のAppleにも、ジョブズの遺産を活かしつつ、ジョブズのいない自由も活かしていけば、チャンスはあるんじゃないですかね。今までのAppleはジョブズがいないことを悲劇としかとらえていなかったのでは？かつては追い出したくせに、今はジョブズがいれば…ってどこかで思ってる、それがダメなんですよね。ジョブズならどうしたか…なんて考えるのはバカげたことです。喪に服すのは終わりにして、もっとヒャッハーしてくれませんかね。そりゃ、死なれちゃ、逆らうにも逆らえないのも分かりますがね。ちゅうわけで、AppleにもAppleユーザ達にも幸あれと、祈っておいて終わりにしましょう。
